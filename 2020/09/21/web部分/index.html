数据库基本概念

  • 特点：
    • 持久化存储数据(表名数据库一个文件系统)
    • 方便存储和管理数据
    • 使用了统一的方式来操作数据库 SQL
  • net stop/start mysql 可以关闭或者打开mysql服务 要在管理员的cmd下运行

    mysql数据库部分

  • SQL基本概念
    • 结构化查询语言：定义了操作所有关系型数据库的规则。每一种数据库存在不一样的地方，被称为”方言
  • sql通用语句
    • sql可以以单行或者多行书写 以分号结尾
    • sql语句不区分大小写
    • 单行注释 – (需要加空格) 或者#
    • 多行注释 /* */
  • sql分类
    • DDL：数据定义语言
      • 操作数据库和表
    • DML：数据操作语言
      • 增删改表中的数据
    • DQL：数据查询语言
      • 查询语句
    • DCL：数据库控制语言
      • 进行授权操作

        DDL操作数据库

  • 操作数据库: CRUD
    • C 创建
      • create database 数据库名; 创建数据库
      • create database if not exists 数据库名; 如果不存在就创建
      • create database 数据库名 character set gbk;创建数据库 设定编码为GBK
    • R 查询
      • show databases;查看所有数据库
      • show create database 数据库名称; 查看创建数据库的语句
    • U 修改
      • alter database 数据库名 character set 字符集名称; 修改数据库的字符集
    • D 删除
      • drop database 数据库名; 删除数据库名
      • drop database if exists 数据库名; 如果存在就删除
    • 用数据库
      • select database(); 查看当前使用的数据库
      • use 数据库名;使用数据库
  • 数据类型
    • double(5,2)表示一共5位 小数点后有两位
  • DML操作表
    • 创建
      • create table 表名(列名1 数据类型1, 列名2 数据类型2, ….);
      • create table 表名 like 表名2; 复制表2
    • 查询
      • show tables; 查询某个所有表名称
      • desc 表名; 查询表结构
      • show create table 表名; 查看表结构
    • 修改
      • 修改表名
        • alter table 表名 rename to 新表名;
      • 修改表的字符集：alter table 表名 character set utf8;
      • 添加一列: alter table 表名 add 列名 数据类型;
      • 修改列名称和类型: alter table 表名 change 旧列名 新列名 新类型
      • 只修改列类型:alter table 表名 modify 列名 新类型
      • 删除列：alter table 表名 drop 列名;
    • 删除
      • drop table(if exists) 表名; 删除表名
      • TRUNCATRE table 表名; 删除表然后创建一个一样的空表

        DML增删改查表中的数据

  • 添加数据
    • insert into 表名(列名1,列名2…) values(值1, 值2….);
    • 列名和值要一一对应
    • 如果表名后不定义列名 默认给所有值添加值
  • 删除数据
    • delete from 表名 [where条件];删除某条数据
    • 如果没条件就删除所有数据 一般使用 TRUNCATRE
  • 修改数据
    • update 表名 set 列名1=值1, 列名2=值2,….[where条件];

      DQL 查询

  • 查询所有: select * from 表名; 查询所有记录
  • select 字段列表 from 表名 [where条件] [group分组条件] [having分组后的条件] [order by 排序] [limit 分页];
  • select去重
    • select distinct ….;
  • select 中技巧
    • select name,math+english from student;
    • 如果有null 那么计算结果都是null、
    • select name,math+IFNULL(english,0) from student; 就可以了
    • select name,math+IFNULL(english,0) as 总分 from student; 起别名
  • 条件查询
    • where字句后跟条件

    • where 字段名 in (xxxx);
    • 查询NULL不可以用等号判断 只能用 is和not is
  • 模糊查询
    • 用like
    • _表示单个字符 %表示多个字符
    • select * from student where name like ‘马%’;
  • 排序查询：order by
    • order by 排序字段1 方式1，字段2 排序方式2;
    • ASC：升序 也是默认的; DESC: 降序
  • 聚合函数：将一列数据作为一个整体 进行一列的计算（都是排除了非空的元素）
    • count 计算个数
      • count(*) 一行有一个不为null就算进去
    • max计算最大值
    • min计算最小值
    • sum计算和
    • avg计算平均值
    • 如果不想放弃null 可以 select count(ifnull(english,0)) form student;这样
  • 分组查询
    • group By 分组字段;
    • 注意：
      • 分组后查询的字段：分组字段 聚合函数
      • where在分组之前限定 having在分组之后限定
      • where不可进行聚合函数的判断 havin可以进行聚合语句的判断
  • 分页查询
    • limit 开始的索引,每页查询的条数;
    • 如果最后一页 显示不够了 有多少 显示多少
    • 分页数据库是一个“方言”

数据库约束

  • 对表中的数据进行限定 保证数据的正确性 有效性和完整性
  • 在创建表时，列名后添加即可
  • 主键约束: primary key
    • 非空且唯一 一张表只有一个字段作为主键
    • 主键就是表中记录的唯一表示
    • 删除主键：alter table 表名 drop primary key;
    • 创建表后添加: alter table 表名 modify 列名 int primary key;
  • 非空约束：not null
    • 创建表后增加:alter table 表名 modify name varchar(20) not null;
    • 删除非空约束:alter table 表名 modify name varchar(20); 不写not null即可
  • 自动增长:auto_increment
    • 给一个数值类型的列 让他自动增长 一般配合主键使用
    • 添加：alter table 表名 modify 列名 类型 auto_increament;
    • 删除：alter table 表名 modify 列名 类型;
    • 自动增长后可以在添加值的时候给null
    • 插入数据的时候，只和上一条数据后关系。如果插入数据的时候跳过一部分数，以后的数据也会从当前开始增长
    • 删除一部分数据后
  • 唯一约束：unqiue
    • 不重复。null也是不可以重复
    • 删除唯一约束:alter table 表名 Drop index 列名;
    • 创建表后添加唯一约束:alter table 表名 modiry 列名 unique;
  • 外键约束：foreign key
    • 外键可为null 但是不能是另一个表不存在的
    • 级联更新：更新一个表字段 另一个有关系的也会更新
    • 级联删除：删除一个表的字段 外键对应的字段也会更新
    • 好处是方便 弊端是误操作

      多表关系

  • 多表之间的关系
    • 一对一 : 人和身份号
    • 一对多：部门和员工
    • 多对多：学生和课程

      范式

  • 设计关心数据库时，遵从不同的规范要求，设计出合理的关系型数据库，这些不同的规范要求被称为不同的范式，各种范式呈递归规范，越高的范式数据库冗余
    基本概念
    - 函数依赖：也就是通过A的属性值(或者属性组) 可以得到唯一B的属性值，则B依赖于A
    - 学号->姓名 (学号,课程)->分数
- 完全函数依赖:如果A是一个属性组 则B属性值的确定需要依赖A属性组中 所有的属性值。
    - （学号，课程名称）->分数
- 部分函数依赖：A是一个属性组 如果B的属性值只需要依赖A中的部分属性
    - (学号，课程名称)->姓名
- 传递函数依赖：A->B,B->C; 如果通过A属性（属性组）的值可以确定唯一B的值 在通过B属性（属性组）的值可以判断唯一的C，则C传递依赖A
    - 学号->系名 系名->系主任
- 码：如果一张表中 一个属性或者属性组 被其他所有属性完全依赖就被称为该表的码
- 主属性：码属性组中的所有属性
- 非主属性：出国码属性组的属性
  • 第一范式
    • 每一列都是不可分割的原子数据项
  • 第二范式
    • 在1NF的基础上，非码属性必须完全依赖候选码(在1NF上消除非主属性对主码的部分函数依赖)
  • 第三范式
    • 在2NF的基础上 任何非主属性不依赖于其他非主属性(在2NF技术上消除传递依赖)

      多表查询

  • 查询语法
    • select 列名列表 from 表名列表 where…;
  • 多表查询的结果不过滤叫笛卡尔积
  • 内连接查询
    • 隐式内连接：使用where条件
    • 显式内连接(inner可以省略)
  • 外连接查询
    • 左外连接
      • 语法：select 字段 from 表1 left [outer] join 表2 on 条件;
      • 左外连接查询的是左表所有数据以及其所有交集部分
    • 右外连接
      • 和左外连接相对的
  • 子查询
    • 查询里面嵌套查询 称嵌套查询为子查询
    • 子查询的结果是单行单列的
      • 子查询作为条件 使用运算符进行判断
    • 子查询的结构是单行多列的
      • 也可作为运算符判断 使用运算符”in”
    • 子查询的结果是多行多列的
      • 可以作为一个虚拟表 进行表的查询

        事务

  • 概念:如果一个包含多个步骤的业务操作，被事务管理，那么这些操作要么同时成功，要么同时失败。
  • 开启事务：start transaction;
  • 提交事务：commit
  • 回滚事务：rollback;在哪开的事务 就回到开启事务的时候
  • 在mysql数据库中 事务默认自动提交
    • 一条DML（增删改）会自动提交一次事务
  • 查看事务的默认提交方式:select @@autocommit;
    • 1代表自动提交
    • 0代表手动提交
  • 修改默认提交方式 set@@autocommit =0;

    事务的四大特征

  • 原子性:是不可分割的最小操作单位，要么同时成功 要么同时失败
  • 持久性：如果事务一旦提交或者回滚，数据库表的诗句会永久的更新
  • 隔离性：多个事物之间相互独立
  • 一致性：事务操作的前后数据总量不变

    事务的隔离级别（了解）

  • 多个事务之间是隔离的 但是如果多个事务操作同一批数据会引发问题，也就需要设定不同对隔离级别
  • 存在问题：
    • 脏读：一个事务读取到另一个事务中没有提交的数据
    • 不可重复读(虚读)：在同一个事务中 两次读取到的数据不一样(读两次一样的数据 间隔中另一个操作数据的事务回滚了 导致俩次结果不一样)
    • 幻读：一个事务操作(DML)数据表总所有的记录，另一个事务添加了一条数据，则第一个事务查询不到自己的修改。
  • 隔离级别：
    • read uncommited:读未提交
      • 产生的问题：脏读,不可重复度,幻读
    • read commited:读已提交
      • 产生问题:不可重复度,幻读
    • repeatable read（mysql默认的）：可重复度
      • 产生问题: 幻读
    • serializble:串行化
      • 可解决所有问题
      • 上锁 一行同时只能被一个事务操作
  • 隔离级别从小到大 安全性越来越高 但是效率越来越低
  • 查询隔离级别：select @@tx_isolation
  • 设置隔离级别 set global transaction isolation level 级别字符串
    • 重新打开才会生效

      DCL

  • DCL: 管理用户的 授权
  • DBA：数据库管理员
  • 通配符% 表示可以从任意主机访问数据库
  • 先切换到mysql数据库
    • 查询用户
      • select * from USER;
    • 添加用户
      • create user “用户名”@’主机名’ identiiyed by ‘密码’;
    • 删除用户
      • drop user ‘用户名‘@’主机名’
    • 修改用户密码
      • update user set password=Password(“信息吗”) where USER=”用户名”;
      • set password for “用户名”@”主机名”=PASSWORD(“新密码”)
    • 忘记root用户
      • net stop mysql 停止mysql服务
      • mysql –kip-grant-tables然后直接打开新窗口输入 mysql即可，修改密码
      • 打开任务管理器 关闭mysqld.exe

        权限管理

  • 查询权限
    • show grants for “查询的用户”@”主机名”
  • 授予权限
    • grant 权限列表 on 数据库.表名 to “用户”@”主机名”
    • 可以使用all表示所有权限 .表示所有库的所有表
  • 撤销权限
    • revoke 权限列表 on 数据库.表名 from “用户”@”主机名” ;

      JDBC

  • Java DataBase Connectivity：java数据库连接
  • java语言操作数据库
  • JDBC本质：官方定义的一套操作所有关系型数据库的规则，即接口。各个数据库厂商去实现这套接口提供数据库驱动jar包，真正执行的代码是驱动jar包中的实现类

    使用过程

  • 流程
    1. 导入jar包
      • 建立一个目录存入jar包 邮件jar包 add as library
    2. 注册驱动
    3. 获取数据库连接对象 Connection
    4. 定义sql
    5. 获取执行sql对象 Statement
    6. 执行sql 接受返回结果
    7. 处理结果
    8. 释放资源
  • jdk5之后可以不注册驱动 会自动注册驱动
  • 详解各个对象(都是借口 实际用的是jar包里面的实现类):
    • DriverManager:驱动管理对象
      • 注册驱动
        • Class.forName()就是用了DriverManager的registerDriver()方法
      • 获取数据连接
        • getConnection(url,user,password)获取数据库连接
    • Connection：数据库连接对象
      • 获取执行sql的对象
        • Statement createStatement()
        • PreparedStatement prepaseStatement()；
      • 管理事务
        • 开启事务 setAutoCommit()调用该该方法为False 即开启事务
        • 提交事务 commit()
        • 回滚事务 rollback()
    • Statement:执行静态sql对象
      • boolean execute() 执行任意sql 不常用
      • int executeUpdate() 执行DML(增删改)语句 DDL(创/删 表/库)。返回值是影响的行数，也就表名返回值大于0就表示执行成功
      • ResultSet executeQuery():执行DQL(查询)语句
        • 用游标遍历
        • next()游标下移方法
        • getXXX() xxx表示数据类型 参数表示获取第几列 从1开始。也可以穿String表列名称
    • ResultSet:结果集对象
    • PreparedStatement:执行动态sql对象
      • 为了解决sql注入问题
      • 效率更高 一般都用这个
