关系型数据库和非关系型数据库

  • 关系数据库(SQL)很强大，但是无法应付所有逇应用场景。mysql的扩展性差，大数据下IO压力大，表结构更改困难

  • 非关系型(NoSQL)易扩展，大数据量高性能，灵活的数据模型(不需要提前建立数据字段)，高可用

    • 缺点:有可能存储大量重复的信息

      安装和启动

  • sudo apt-get install -y mongodb-org 安装

    • sudo service mongod start 服务端启动
    • sudo service mongod stop 服务端停止
    • sudo service mongod restart 服务端重启

  • 下载后解压安装(省略)

    • sudo mongod – config 配置文件路径
    • /etc/mongod.conf 配置文件位置

  • 客户端启动: mongo

  • 查看帮助 mongo -help

  • 退出 exit或者ctrl+c

    基础命令

  • show databases 查看数据库

  • use xxx 使用xxx数据库(如果数据库不存在 也可以use 相当于创建)

    • 刚创建的数据库 show dbs 不会显示 需要插入数据后显示

  • db 查看当前使用的数据库

  • db.dropDatabase() 删除当前使用的数据库

    集合

  • mongodb 没有表 都是存放在集合中

  • 创建集合

    • 向不存在的集合中插入一个数据，就可以创建

  • 手动创建集合

    • db.createCollection(name,options)
    • 例如 db.createCollection(“stu”)
    • 也可以传入参数 db.createCollection(“sub”,{capped:true,size:10}) capped:true表示设置上限，size表示最大容量
    • 如果超过了 后面的数据会把前面的覆盖(相当队列,把前面的重复)

  • show collections 显示数据库中所有的集合

  • db.集合名.drop() 就删除对应的集合

    能够存储的数据类型

  • Object ID:文档ID

    • 存在数据库的字典就是文档 每个文档(也就是每条信息)都有一个id(默认为_id) 是一个object id
    • 是一个12字节的16位进制数 前4个字节为当前时间戳， 接下来3个是机器的id 接下来2个是mongodb服务进程id最后三个是简单的增量

  • String 字符串 必须是有效的UTF-8

  • Boolean: 储存一个布尔值 true or false (必须小写)

  • object: 字典里面还有一个字典

  • null: 存储null值

  • data: 存储当前的时间或时间UNIX时间类型

  • Timestamp: 时间戳 表示1970-1-1 到现在的总秒数

    集合操作

  • db.集合名.find() 显示集合的所有值

  • db.集合名.insert({“name”:18}) 插入数据

  • db.集合名.save(document)

    • 如果设置_id _id存在就修改 不存在就插入一条新的
    • db.集合名.insert({“_id”:xxx,name”:18}) 插入数据的时候 如果设置_id 就必须和数据库已经存的不一样

  • db.集合名.update(,,{multi:}) 三个参数分别是更新条件(不写就是修改全部) 更新内容 multi是True表示修改所有满足的文档 False就是表示修改一个文档

    • 比如 dp.test100.update({name:’xiang’},{name:”xxxx”}) 把整条记录==替换==为name:xxx 如果就多个键值对被直接删除
    • 如果就想修改文档中一个键值对db.test100.update({name:xxx},{$set:{name:”xiao”}}) 只会修改文档中的name字段 其他的键值对不会动
    • 一个是修改name对应的值，另一个是直接替换掉整个文档
    • multi必须和$ 同时使用

  • db.集合名.remove(,{justOne:) 条件 justOne默认删除多条 true是只删除一条

    • db.test100.remove({name:’xiaohong’}) 删除name为xiaohong的

      高级查询

  • db.集合名.find({“字段名”:”字段值”}) 查询包涵这个字段的集合

    • db.集合名.find({“字段名”:”字段值”}).pretty() 输出结果格式化

  • 比较运算符

    • 等于:默认就是等于
    • 小于: $lt
    • 大于等于: $lte
    • 大于: $Gt
    • 大于等于: $gte
    • 不等于:$ne
    • db.test100.find({name:{$ne:’cy’}}) 就是查询不为name不为‘cy’的集合

  • 范围运算符

    • $in 在某个范围 db.xxx.find({age:{$in:[12,1231,213]}})
    • $nin 不在某个范围内

  • 逻辑运算符

    • and 直接写多个条件就可以了
    • $or db.stu.find({$or:[{age:$gt:18},{gender:false}]}) 查询年龄大于18 或者性别为false的学生

  • 正则表达式

    • db.xxx.find({sku:/^abc/})
    • db.xxx.find({sku:[$regex:”789$”]})

  • 一些方法

    • 方法limit() 读取指定数量的文件
    • 方法skip() 用于跳过指定数量的文档
    • 可以结合使用

  • 自定义查询

    • 查询年龄大于30的学生
    • db.stu.find({
      $where:function(){
      return this.age>30;
      }
      })
    • 设置返回值的字段 设置1就是返回 不设置就不返回 _id 默认会返回 设置0就不返回
    • 不能把非_id的字段设置为0
    • db.stu.find({
      $where:function(){
      return this.age>30;
      }
      },{name:1})

  • 排序

    • db.stu.find().sort({age:1})按照age升序排列
    • db.stu.find().sort({age:-1})降序
    • true比false排再后面

  • 统计个数

    • db.stu.find().count() 统计个数
    • 如果直接 db.stu.count({}) count里面可以写条件

  • 去重

    • db.stu.distinct(“hometown”) [1,2,3] 对hometown去重
    • d.stu.disctinct(“hometown”,{age:{$gt:20}}) 先找大于20的然后在distinct

      数据备份和回复

  • mongodump -h dbhost -d dbanme -o dbdirectory

    • -h 服务器地址也可以指指定端口号
    • -d 需要备份的数据库名称
    • -0 备汛的数据存放位置，此目录中存放着备份出来的数据

  • mongorestore -h dbhost -d dbanme –dir dbdirectory

    • -h 服务器地址
    • -d 需要回复的数据库实例
    • -dir 备份数据放的位置

      聚合

  • 聚合就是聚合管道 每个文档通过多个阶段吧组成的管道 可以对每个阶段的管道进行分组，过滤等功能，最终输出其结果

  • db.集合名.aggregate([{管道:{表达式}}])

  • 管道

    • $group:分组 用于统计结果
    • $match:过滤 输出符合条件的文档
    • $project 修改文档的结果 如重命名 增加 删除字段 创建计算结果
    • $sort: 输入文档排序后输出
    • $limit: 限制聚合管道返回的文档数
    • $skip: 跳过指定数量的文档 并且返回余下的文档

  • 表达式

    • $sum: 求和
    • $avg: 平均值
    • $min: 获取最小值
    • $max: 获取最大值
    • $push: 结果文档插入值到一个数组中
    • $first: 根据资源文档的排序获取第一个文档的数据
    • $last: 获取最后一个

  • 例子

  • group注意点

    • group对应的字典有几个键 结果中就有几个键
    • 分组依旧需要放到”_id”后面
    • 取不同的字段的值 需要使用啊$符号

  • $project 修改输出结果

  • $match 是管道命令 可以把结果交给下一个管道 但是find就不行

  • 去字典中的字典 就 xxx.xxx

  • 可以按照多个键分组

    • 结果会是”{_id:{xxxx}}”

  • 同时按照所有字段分组 就可以达到去重的效果

  • $sort 排序

    • b.stu.aggregate({$sort:{age:1}})
    • 1表示升序 -1 表示降序

  • $limist: db.stu.aggregate({$limit:2})

  • $skip: db.stu.aggregate({$skip:2})

  • $unwind

    • preserveNullAndEmptyArrays 默认是false表示丢弃属性值为空的文档
    • db.t3.aggregate({$unwind:{path:”$size”,preserveNullAndEmptyArrays:True})
      • 比如有有的记录 size没有 或者size的值为空 但是依旧不会被忽略

        索引

  • db.集合名.ensureIndex({xxx:1}) 建立xxx为索引 1表示升序建立索引 -1表示降序

  • 默认情况下_id有一个索引

  • 默认情况下索引的字段的值可以相投

  • 建立唯一索引:db.t1.ensureIndex({“name”:1 },{‘unique’:true})

    • 如果企图插入一条重复的数据会报错
    • 可以利用该特性 进行去重

  • 建立联合索引:db.t1.ensureIndex({name:1,age:1})

    • 建立联合索引唯一 可以去重

  • 查看当前索引:db.t1.getIndexes()

  • 删除索引:db.t1.dropIndex(“索引名称”)
